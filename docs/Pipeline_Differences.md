# DAZZ Retro Camera 渲染管线差异对比（预览 vs 成片）

经过全面重构后，DAZZ Retro Camera 的预览和成片管线已经高度统一，均采用参数驱动的通用 Shader 架构。然而，由于**实时性能要求**和**画质追求**的差异，两条管线在具体 Pass 的数量、顺序和算法实现上仍存在设计上的差异。

以下是预览管线（`CameraGLRenderer`）与成片管线（`CaptureGLProcessor`）的逐 Pass 详细对比。

## 1. 整体管线结构对比

| 特性 | 预览管线 (Preview) | 成片管线 (Capture) | 差异原因 |
| :--- | :--- | :--- | :--- |
| **执行环境** | 实时渲染 (60fps)，直接上屏 | 离屏渲染 (单张)，保存为 JPEG | 性能要求不同 |
| **渲染架构** | 两步渲染 (OES -> FBO -> Screen) | 单步渲染 (Bitmap -> FBO -> Bitmap) | 预览需解决 OES 纹理多次采样撕裂问题 |
| **总 Pass 数量** | 21 个 Pass (含锐化) | 19 个 Pass (无锐化、无 ToneCurve) | 预览包含更多特定相机的实时模拟 |
| **分辨率** | 屏幕分辨率 (如 1080x1440) | 照片原始分辨率 (如 3000x4000) | 像素级效果（如颗粒、模糊）的感受会不同 |

## 2. 逐 Pass 详细对比表

下表列出了所有渲染 Pass 的执行顺序及两端差异。

| 顺序 | 效果名称 (Pass) | 预览管线 (CameraGLRenderer) | 成片管线 (CaptureGLProcessor) | 差异说明 |
| :--- | :--- | :--- | :--- | :--- |
| **0** | **锐化 (Sharpen)** | ✅ 执行 (3x3 高斯模糊 + 差值) | ❌ 不执行 | 预览由于降分辨率通常需要锐化，而成片保留了原图细节，不需要额外锐化。 |
| **1** | **色差 (Chromatic Aberration)** | ✅ 执行 | ✅ 执行 | 算法完全一致。 |
| **2** | **色温与色调 (Temp & Tint)** | ✅ 执行 | ✅ 执行 | 算法完全一致（均除以 1000 映射到 ±0.2）。 |
| **3** | **黑场与白场 (Blacks & Whites)** | ✅ 执行 | ✅ 执行 | 算法完全一致。 |
| **4** | **高光与阴影 (Highlights & Shadows)**| ✅ 执行 | ✅ 执行 | 算法完全一致。 |
| **5** | **对比度 (Contrast)** | ✅ 执行 | ✅ 执行 | 算法完全一致。 |
| **6** | **清晰度 (Clarity)** | ✅ 执行 (3x3 采样) | ✅ 执行 (3x3 采样) | 算法完全一致。 |
| **7** | **饱和度与自然饱和度 (Sat & Vib)** | ✅ 执行 | ✅ 执行 | 算法完全一致。 |
| **8** | **RGB 通道偏移 (Color Bias)** | ✅ 执行 | ✅ 执行 | 算法完全一致。 |
| **9** | **高光光晕 (Bloom)** | ✅ 执行 | ✅ 执行 | 算法完全一致。 |
| **10** | **高光辉光 (Halation)** | ✅ 执行 | ✅ 执行 | 算法完全一致。 |
| **11** | **高光滚落 (Highlight Rolloff)** | ✅ 执行 (`applyHighlightRolloffPreview`) | ✅ 执行 (`applyHighlightRolloff`) | **算法不同**：预览使用基于二次多项式的平滑压缩，成片使用基于线性插值的压缩。 |
| **12** | **中心增亮 (Center Gain)** | ✅ 执行 (`ccdCenterEdge` 合并处理) | ✅ 执行 (`applyCenterGain` 独立处理) | **实现不同**：预览将 Center Gain 和 Edge Falloff 合并为一个 Pass 计算，成片拆分为两个独立函数。 |
| **13** | **肤色保护 (Skin Protect)** | ✅ 执行 (`ccdSkinProtect`) | ✅ 执行 (`applySkinProtect`) | **算法不同**：预览使用简化的 HSL 转换和肤色遮罩，成片使用更精细的 HSV 转换和更严格的色相范围限制 (0~50度)。 |
| **14** | **边缘衰减 (Edge Falloff)** | ✅ 执行 (`ccdCenterEdge` 合并处理) | ✅ 执行 (`applyEdgeFalloff` 独立处理) | 见 Pass 12。 |
| **15** | **曝光波动 (Exposure Variation)** | ✅ 执行 (基于时间的随机噪声) | ❌ 不执行 | **仅预览有**：这是一种动态的画面闪烁效果，仅在视频或实时预览中有意义，静态成片不需要。 |
| **16** | **角落偏色 (Corner Warm Shift)** | ✅ 执行 (`ccdCornerWarm`) | ✅ 执行 (`applyCornerWarm`) | 算法基本一致。 |
| **17** | **显影柔化 (Development Softness)** | ✅ 执行 (十字形 4 点采样模糊) | ❌ 不执行 | **仅预览有**：可能因为成片分辨率太高，4点采样模糊效果不明显，因此在成片管线中被舍弃。 |
| **18** | **高光滚落 2 (Highlight Rolloff 2)**| ✅ 执行 (`ccdHighlightRolloff`) | ❌ 不执行 | **仅预览有**：FXN-R 相机专属的二次高光压缩，配合 Tone Curve 使用。 |
| **19** | **色调曲线 (Tone Curve)** | ✅ 执行 (`fxnrToneCurve`) | ❌ 不执行 | **仅预览有**：FXN-R 相机专属的分段线性曲线，成片管线中未实现此逻辑。 |
| **20** | **化学不规则感 (Chemical Irregularity)**| ❌ 不执行 | ✅ 执行 (`applyChemicalIrregularity`) | **仅成片有**：基于 hash 噪声的红绿通道微小扰动，模拟胶片化学反应的不均匀性。 |
| **21** | **相纸纹理 (Paper Texture)** | ✅ 执行 (基于 random 噪声) | ✅ 执行 (基于 hash 噪声) | **算法不同**：预览使用高频白噪声模拟，成片使用基于 UV 坐标哈希的低频噪声模拟，视觉质感略有不同。 |
| **22** | **胶片颗粒 (Film Grain)** | ✅ 执行 (基于 random 噪声) | ✅ 执行 (基于 hash 噪声) | **算法不同**：预览使用高频白噪声，成片使用基于时间种子的哈希噪声。 |
| **23** | **数字噪点 (Digital Noise)** | ✅ 执行 (明度+色度独立计算) | ✅ 执行 (统一计算) | **算法不同**：预览将噪点拆分为明度噪点和 RGB 色度噪点独立计算，成片使用统一的哈希噪声叠加。 |
| **24** | **暗角 (Vignette)** | ✅ 执行 (`vignetteEffect`) | ✅ 执行 (`applyVignette`) | **算法不同**：预览使用点积二次衰减，成片使用 `smoothstep` 线性衰减，且成片会额外叠加镜头暗角 (`uLensVignette`)。 |

## 3. 核心差异总结

1. **动态效果的取舍**：预览管线包含 `Exposure Variation`（曝光波动）等随时间变化的动态效果，而成片作为静态图片去掉了这些计算。
2. **FXN-R 相机专属逻辑**：预览管线保留了 `Highlight Rolloff 2` 和 `Tone Curve`，这是专门为 FXN-R 相机设计的曲线映射；而成片管线中没有这些逻辑，这会导致 FXN-R 相机的预览和成片在对比度和色彩映射上存在肉眼可见的差异。
3. **噪声与纹理的生成算法**：
   - 预览管线使用简单的 `fract(sin(dot(...)))` 伪随机数生成高频白噪声。
   - 成片管线使用更复杂的 `hash` 函数生成低频噪声，质感更接近真实的胶片颗粒和相纸纹理。
4. **肤色保护的精度**：成片管线的肤色提取（HSV 转换）比预览管线（HSL 转换）更精确，对红色的限制更严格，因此成片中的人物肤色通常比预览更自然、更不容易过曝。
